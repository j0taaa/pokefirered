## Hidden Item Text Flow

The hidden item text flow in FireRed uses extended control codes to switch text color
mid-string, then advances the page with CHAR_PROMPT_CLEAR. The decompBrailleText module
covers this flow through the following control code sequence:

1. EXT_CTRL_CODE_BEGIN + EXT_CTRL_CODE_COLOR_HIGHLIGHT_SHADOW (0x04) - reads 3 params
   (fg, bg, shadow) and generates a font half-row lookup table
2. Glyph printing advances currentX by glyphWidth + letterSpacing
3. EXT_CTRL_CODE_BEGIN + EXT_CTRL_CODE_COLOR (0x01) - reads 1 param, updates fgColor
4. CHAR_PROMPT_CLEAR (0xfb) - enters CLEAR state, fills window with bgColor, resets cursor
5. EOS (0xff) - terminates the printer

The "put the PEARL" sequence specifically uses:
- Color/highlight/shadow to set item name color
- Shift codes to position text
- Fill window to clear before the next page

## Control Code Coverage

All FireRed text control codes are covered by the decompBrailleText module:
- COLOR (0x01), HIGHLIGHT (0x02), SHADOW (0x03), COLOR_HIGHLIGHT_SHADOW (0x04)
- PALETTE (0x05), FONT (0x06), RESET_FONT (0x07)
- PAUSE (0x08), PAUSE_UNTIL_PRESS (0x09), WAIT_SE (0x0a)
- PLAY_BGM (0x0b), ESCAPE (0x0c), SHIFT_RIGHT (0x0d), SHIFT_DOWN (0x0e)
- FILL_WINDOW (0x0f), PLAY_SE (0x10)
- CHAR_NEWLINE (0xfe), CHAR_PROMPT_CLEAR (0xfb), CHAR_PROMPT_SCROLL (0xfa)
- CHAR_EXTRA_SYMBOL (0xf9), CHAR_KEYPAD_ICON (0xf8), PLACEHOLDER_BEGIN (0xfd)

## Glyph Metrics

Braille glyph metrics:
- Width: always 16 pixels (matching GetGlyphWidth_Braille)
- Height: always 16 pixels
- Source offsets: [base, base+0x8, base+0x80, base+0x88] where base = 0x100 * (glyph/8) + 0x10 * (glyph%8)
- Letter spacing and line spacing advance cursor position
