Task 11: Menu Parity Implementation Summary

## Game Logic Files Created

### ts-game/src/game/pokemonCenter.ts
- PokemonCenterState with stages: idle, nurseWelcome, nurseTakePokemon, healing, nurseRestored, nurseGoodbye, pcAccess, pcExit
- Nurse dialogue flow: welcome -> takePokemon -> healing animation -> restored -> goodbye
- Healing animation with configurable duration (120 frames)
- PC access open/close flow
- Decline path returns to idle with goodbye text
- All decomp-correct nurse messages preserved

### ts-game/src/game/namingScreen.ts
- Wraps decompNamingScreen with InputSnapshot-based step function
- Supports player (7 chars), rival (7 chars), Pokemon nickname (10 chars), box (8 chars)
- Keyboard navigation with d-pad, character entry, page swap, backspace, OK
- Cancel (B) returns unconfirmed result
- Re-exports keyboard chars, column count, key role, text entry position for UI

### ts-game/src/game/titleScreen.ts
- TitleScreenState with stages: title, intro, newGame, continue, options, credits
- Title menu with CONTINUE/NEW GAME/OPTION (has save) or NEW GAME/OPTION (no save)
- Intro timer with configurable duration
- Credits scroll with configurable speed
- Keyboard navigation matching decomp (up/down, A/start to select, B to cancel)

### ts-game/src/game/pcStorage.ts
- PcStorageState with deposit/withdraw/move operations
- Deposit: validates party size > 1, box capacity, moves Pokemon from party to current box
- Withdraw: validates party size < 6, moves Pokemon from box to party
- Move between boxes: validates source/destination box IDs and capacity
- Box switching with wrap-around (14 boxes)
- Box renaming (8 char limit)
- getCurrentBoxSlots/getPartySlots for UI rendering
- PC action labels matching decomp (WITHDRAW, DEPOSIT, MOVE, MOVE ITEMS, CANCEL)
